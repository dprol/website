Bitbucket

We will be using Bitbucket for lab assignments in this class. I'll explain what Bitbucket is, why it's useful, and how to use it, in the first class session, but it's basically a cloud service for storing files, sort of like Dropbox.

The difference between Bitbucket and Dropbox is that Dropbox handles things more or less automatically, whereas Bitbucket, which uses the version control system Git, does only what you tell it to do. Git is a whole lot more flexible, durable, and scalable than Dropbox, and version control systems are pretty much ubiquitous in software development, which is why I'll be teaching you how to use it in this course.

All information you upload to Bitbucket in this class will be private. Only you and the people with whom you share it (which will include me, of course) will be able to view that information. If your parents have any restrictions or concerns about creating online accounts, make sure you have their approval before you sign up.

In order to use Bitbucket, you'll first need to create an account. You will be asked for your name, a username, a password, and your email address. The name field is optional, so don't fill it out if you don't want to. After you've created an account, you're done! There's nothing else you need to do with Bitbucket before class starts.
